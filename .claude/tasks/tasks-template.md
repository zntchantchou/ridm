# These are the rules all tasks within this "tasks" folder must follow

## Naming Convention

Format: ###-the-task-name.md
Example: 001-setup-eslint.md

## Sections

- Quick task description

Very briefly explain what the task will achieve

- Task checklist

A checklist of steps to take for that task.
After each task, the task list steps must be checked
If a clarification is needed add for a task [STILL UNCERTAIN]
Ask for review of all uncertainties before proceeding

- File structure modifications

The Task will show all the paths that will need to be updated.
Group them by BEFORE and AFTER for a readable report.

- Wordcount

Include a wordcount section at the end. The word count will show the number of characters added while implementing this task. This number can be negative of course.

# Writing style

Wording must be concise, use simple phrasing.
