# These are the rules all tasks within this "refactor" folder must follow

## Guidelines File

Argument $1 is the folder name as a three digits number (incremental) where you will create the refactor task and look for the guidelines file.
Read any guidelines file present in the directory for instructions before coming up with the refactor 'task'.
Specific instructions can be passed (optional) by using the second argument: $2. It designates the file to look for the guidelines (\*.guidelines.md) file for this type of task.
Also look at any updates in the code before creating the task.

## Naming Convention

Format: ###-refactor-[the-task-name].md
Example: 001-refactor-stepper-registration.md

## End of task information section

In the task do provide a BEFORE and AFTER view of the folders and files to be created
